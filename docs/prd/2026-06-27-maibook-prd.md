 ## 1. 产品概述
 
 Maibook 是一款面向学术论文阅读的本地桌面应用，基于对 Moonlight Chrome 扩展（v3.0.4）的反向工程分析进行复现与扩展。它将 Moonlight 的核心能力从浏览器扩展形态迁移为独立桌面应用，并引入创新的 Skill Hub 机制，实现 AI 能力的可扩展管理。
 
 **核心理念：** 专注本地、用户掌控数据、AI 能力可扩展、为未来多端同步预留架构。
 
 ---
 
 ## 2. 技术栈
 
 | 层级 | 技术选型 | 说明 |
 |-----|---------|------|
 | 桌面框架 | **Tauri** | 跨平台桌面框架，Rust 核心 + Web 前端 |
 | 前端框架 | **React** | 构建 UI 组件和交互面板 |
 | PDF 引擎 | **pdf.js** | 深度定制的 PDF 渲染引擎 |
 | 本地存储 | **SQLite** | 结构化数据存储 |
 | 配置格式 | **TOML** | 人类可读的配置文件 |
 | AI 接口 | **OpenAI 兼容 API** | 支持 OpenAI、DeepSeek、通义千问等 |
 | 打包格式 | **ZIP (.maibook)** | 数据导出/导入便携格式 |
 
 ---
 
 ## 3. 系统架构
 
 三层架构：Tauri Shell（Rust 后端 + React 前端）+ 用户本地文件系统 + OpenAI 兼容 API。
 
 Rust 层负责文件系统访问、SQLite 存储、配置管理、导出/导入。
 React 前端负责 PDF 阅读器、AI 对话、标注引擎、翻译、文献库、设置。
 
 ---
 
 ## 4. 功能模块
 
 ### 4.1 PDF 阅读器（P0）
 
 基于 pdf.js 深度定制，作为应用的核心容器。
 
 **核心功能：** PDF 渲染、文件打开（拖拽/菜单/URL）、页面导航、缩放、布局模式（单页/双页/连续）、文本选择、全文搜索、书签、大纲提取、打印、页面旋转。
 
 **界面：** 左侧可折叠栏（缩略图/大纲/书签 Tab 切换）+ 中央 pdf.js 主画布 + 右侧可折叠栏（AI 对话/翻译/标注索引）。顶部固定工具栏包含文件、缩放、布局、搜索、视图、设置等操作。
 
 ### 4.2 AI 引擎（P0）
 
 所有 AI 功能的后端驱动，通过 OpenAI 兼容 API 实现。
 
 **能力矩阵：** AI 对话（多轮）、文本解释、公式解释、图像解释、表格分析、摘要生成、大纲生成、测验生成。
 
 **配置：** API Key、API 端点 URL（默认 openai，可改为 DeepSeek/通义等）、模型选择、超时时间。
 
 ### 4.3 AI Skill Hub（P0 创新功能）
 
 Maibook 相对于 Moonlight 的核心创新，将 AI 能力抽象为可扩展的技能体系。
 
 **核心概念：**
 - Skill Hub 是一个可配置的 URL，返回技能市场目录
 - 全部技能分为**已安装**和**未安装**两部分
 - AI 引擎**只能**从已安装列表中自动调用和编排技能
 - 用户在设置中浏览全部技能，一键安装/卸载
 
 **Hub 格式：** 支持 Markdown、JSON 或纯文本，AI 引擎自动解析。
 
 **AI 编排流程：** 用户触发功能 → AI 判断场景 → AI 从已安装技能列表中推荐 → 以上下文菜单呈现 → 用户选择调用。
 
 **管理界面：** Hub URL 配置、刷新、已安装技能列表（启用/禁用）、全部技能浏览（含安装/卸载）。
 
 ### 4.4 标注系统（P0）
 
 **功能：** 文本高亮（4 色预设 + 自定义）、批注评论、笔记（关联论文位置）、画笔绘制（颜色/粗细/可擦除）、标注索引面板（筛选/跳转）。
 
 **交互：** 选中文本 → 浮动工具栏（高亮/翻译/解释/复制/更多）。点击高亮区域 → 操作菜单（编辑评论/修改颜色/删除/复制引用）。
 
 **存储：** JSON 结构化记录，主库 SQLite + annotations/ 目录 JSON 备份。
 
 ### 4.5 翻译功能（P1，默认预装技能）
 
 通过 AI API 驱动，两种模式：
 - **内联翻译**：选中文本 → 弹出翻译气泡，原文+译文对照
 - **整页翻译**：工具栏/右键触发，半透明层叠在原文上方
 
 配置：目标语言、跟随全局 AI 模型、预览长度限制。
 
 ### 4.6 文献管理（P0）
 
 **功能：** 论文库（列表/网格视图）、导入（拖拽/对话框/监视文件夹）、AI 元数据提取（标题/作者/期刊/摘要/DOI）、标签分类、分组管理、搜索、阅读状态（已读/在读/未读）、引用导出（BibTeX/RIS）、最近打开与续读、批量操作。
 
 **界面：** 左侧栏（分类/标签/分组）+ 右侧论文列表（搜索栏 + 筛选器 + 卡片列表）。
 
 **元数据提取：** 导入 PDF → AI 读取元数据 → 不完整则 AI 从首页识别 → 利用 DOI 补充 → 用户可编辑确认。

 ## 5. 数据存储
 
 ### 5.1 目录结构
 
 ```
 maibook/
 ├── config.toml         # 全局配置（API Key、语言、Hub URL 等）
 ├── maibook.db          # SQLite 数据库
 ├── papers/             # PDF 源文件
 ├── skills/             # 已安装技能
 │   └── installed.json
 ├── annotations/        # 标注 JSON 备份
 └── backups/            # 导出的 .maibook 包
 ```
 
 ### 5.2 导出/导入
 
 导出为 `.maibook` ZIP 包，包含 export.json、配置（可选 Key）、SQLite 快照、PDF（可选）、标注备份。
 
 导入策略：合并 / 覆盖 / 手动指定。
 
 ### 5.3 多端同步预留
 
 目录预留 `.sync/`，Rust 层预留 SyncProvider trait（v2+ 实现）。

 ## 9. 风险
 
 - pdf.js 在 Tauri WebView 环境（WebView2）的兼容性需验证
 - AI API 依赖用户自备 Key，体验取决于模型和额度
 - Skill Hub URL 失效时需优雅降级
